# Platform UX and role-based views

## Overview

The MASSIV+ platform presents different views and workflows depending on the user's role. Each persona has a distinct primary task and different levels of technical capability.

## Role-based views

### Large corporation (Ronja)
Primary task: supplier data discovery, hotspot analysis, Scope 3 reporting

Key views:
- Supplier map — which suppliers have shared data, which haven't
- Portfolio view — aggregated Scope 3 across supply chain tiers
- Hotspot analysis — ranked by emissions contribution
- CSRD export — pre-formatted data for statutory reporting

### Large supplier (Anna)
Primary task: manage both inbound (from own suppliers) and outbound (to customers) data flows

Key views:
- Incoming data dashboard — status of data from Anna's own suppliers
- Outgoing sharing settings — who gets to see what, and under what conditions
- PCF calculator — calculate product carbon footprint from input data

### SME supplier (Erik)
Primary task: collect and structure own Scope 1–2 data; share with customers

Key views:
- AI-guided data collection wizard
- Data overview — what has been collected, what is missing
- Sharing dashboard — which customers are requesting data and what's been shared

### Small supplier (Nisse)
Primary task: get from "I have some energy bills" to "my data is shared" with minimal friction

Key views:
- Single-screen upload and guidance flow
- Confirmation that data has been received and shared
